# Skill: Kraken Lead Intelligence Engine

## 1. Skill Type

Coding Agent Skill governing the construction and modification of the Kraken Lead Intelligence Engine.

---

## 2. Project Objective

Build a stateless evaluation engine that extracts business presence from a public URL and selects exactly ONE primary service from `services/services.json`.

---

## 3. Core Constraints & Input Rules

- **Input**: Single URL only (`{"url": "https://..."}`).
- **Extraction**: Fetch public HTML, extract text only, ignore assets/scripts.
- **Limits**: Truncate content for LLM token limits; Single page only; Stateless.
- **Prohibited**: No logins, paywall bypassing, private data scraping, headless browsers (unless approved), or multi-page crawling.

---

## 4. Architecture & File Responsibilities

Structure must remain:
- `extractor.py`: URL fetching, HTML cleaning, structured plain text return.
- `evaluator.py`: Load `services.json`, prompt injection, LLM interaction, JSON enforcement.
- `main.py`: Entry point; orchestrates extractor and evaluator.
- `coda_client.py`: Coda table integration and result mapping.
- `prompts/system_prompt.md`: Canonical LLM instructions.
- `services/services.json`: Service source of truth.

---

## 5. Security & Environment

- **Setup**: Initialize Git, `.gitignore`, and `.env` before first commit.
- **Environment**: Load variables using `python-dotenv`. Required: `GROQ_API_KEY`, `CODA_API_TOKEN`, `CODA_DOC_ID`, `CODA_TABLE_ID`.
- **Validation**: No hardcoded credentials. Stop execution if variables are missing.

---

## 6. Logic & Output Rules

- **Evaluation**: Select exactly ONE service; No hallucinations; Strict JSON output.
- **Validation**: Validate LLM output against `services.json`. Retry once on failure, then error.
- **Output Schema**:
  ```json
  {
    "business_type": "",
    "primary_service": "",
    "fit_score": 0-100,
    "reasoning": "",
    "outreach_angle": ""
  }
  ```
- **Determinism**: Low temperature, consistent structure, minimal randomness.

---

## 7. Performance & Operational Requirements

- **Performance (Strict)**: 
  - URL Fetch: < 5s | HTML Parse: < 2s | LLM Call: < 10s | Coda Insert: < 5s.
  - Total latency: < 20s. Fail if exceeded.
- **Error Handling**: No silent failures. Retry once for network, LLM, or JSON errors; raise controlled exceptions.
- **Logging**: INFO level. Log URL, service, score, Coda result, and errors. No sensitive data.

---

## 8. Development Standards

- **Code Quality**: PEP 8, type hints, docstrings, clean separation of concerns.
- **Dependencies**: `requests`, `beautifulsoup4`, `python-dotenv`, `groq` only.
- **Maintenance**: Python 3.10+; keep dependencies, prompts, and services updated.
- **Anti-Drift**: Reject feature expansion (async, multi-page, caching, etc.) without prior `skill.md` update.

---

## 9. Verification & Documentation

- **Testing**: Validate format, emptiness, non-localhost, non-internal, and non-file URLs. Test integration, error handling, and environment loading.
- **Docs**: Create `README.md` with setup, usage, `.env` guide, error guide, and testing instructions.
- **VCS**: Tag initial commit as `v1.0.0`; ensure `.env` is ignored.
